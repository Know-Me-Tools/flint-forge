# Goals — p12-v1-release

## Phase Summary

Tag `v1.0.0` — the first semantically stable release of Flint Forge. The
platform exits phase 11 with all API surfaces documented, all SDKs at 1.0.0,
Dockerfile secrets fully wired, and a CI CVE gate. The only remaining
prerequisite is replacing the aspirational k6 thresholds with measured values
from a live staging run.

This is a narrow, 2-change phase. Estimated scope: 1 session once staging is live.

Seeded from: `p11-api-stability/reflection.md` → "Recommended Next Phase"

---

## Changes (2 planned)

### P0 — Must ship

- **G1 — p12-c001-k6-measure:**
  Replace aspirational k6 thresholds in `perf/k6/regression.js` with values
  measured against a live staging stack.
  - Run `perf/k6/health.js`, `components.js`, and `mcp_tools.js` against staging
  - Record P50/P95/P99 (and throughput at 10 VUs / 30 s) in `docs/performance.md`
    baseline table
  - Update `perf/k6/regression.js` thresholds: `threshold = ceil(measured_p99 * 1.20)`
    rounded to the nearest 5 ms
  - Set `BASELINE_DATE` and `BASELINE_SOURCE` constants in `regression.js`
  - Run `regression.js` against staging and confirm exit 0
  - Gate: `regression.js` exits 0 against staging

  **Blocker:** Requires a running staging stack
  (`docker compose -f docker-compose.yml -f docker-compose.staging.yml up -d`
  on a cloud host) and a valid staging JWT (`scripts/mint_smoke_token.sh`).

  **Fallback if staging unavailable:** Tag `v1.0.0` with aspirational thresholds
  and document that baselines are TBD in the release notes. The regression gate
  is `workflow_dispatch`-only and not a required CI gate.

- **G2 — p12-c002-v1-tag:**
  Create the `v1.0.0` release.
  - Bump `[workspace.package] version` from `0.10.0` to `1.0.0` in `Cargo.toml`
  - Update `@flint/react` `package.json` (already `1.0.0` — no change needed)
  - Update `flint_genui` `pubspec.yaml` (already `1.0.0` — no change needed)
  - Generate `CHANGELOG.md` section for `v1.0.0` via `git cliff`
  - Commit: `chore(release): v1.0.0`
  - Tag: `git tag -a v1.0.0 -m "Flint Forge v1.0.0 — first stable API release"`
  - Push tag + main
  - Create GitHub Release via `gh release create`
  - Gate: tag exists; GitHub Release visible at `/releases/tag/v1.0.0`

  **Dependency:** Should follow G1 when staging is available, so the release
  notes can cite measured performance baselines. Can run without G1 if staging
  is not available (document k6 TBD in release notes).
