# Goals — p14-v1.1.0

## Phase Summary

First feature cycle after v1.0.0. Delivers the five highest-value items from
`docs/ROADMAP.md`: sqlx Prometheus integration (fixes the Grafana panel +
unblocks `cargo update` for `generic-array`), a Kiln guest Rust SDK for skill
authors, A2UI component hot-reload, staging JWT rotation automation, and
per-function Kiln invocation metrics.

Seeded from: `p13-continuous-operations/reflection.md` → `docs/ROADMAP.md`

---

## Changes (5 planned)

### P0 — Must ship (unblocks existing infrastructure)

- **G1 — p14-c001-sqlx-prometheus:**
  Emit sqlx pool metrics so the Grafana DB connections panel produces data
  and the `HighDbConnections` alert fires on real values.
  - Investigate `sqlx 0.9` upgrade path (needed to unblock `generic-array`
    in `cargo update`; pgvector `Encode`/`Type` traits must unify)
  - Add pool metrics: `sqlx_pool_connections_open`, `sqlx_pool_connections_idle`,
    `sqlx_pool_acquire_total`
  - Wire metrics into the existing `axum-prometheus` recorder in `fdb-gateway`
  - Verify Grafana panel 4 now shows data
  - Gate: `cargo clippy + cargo test + cargo audit` clean; `cargo update` succeeds without breaking pgvector

### P1 — Should ship (high-value features)

- **G2 — p14-c002-kiln-guest-sdk:**
  Create `crates/flint-skill/` — a Rust helper crate targeting `flint:host@0.1.0`.
  - Typed wrappers around WIT bindings: `db::query`, `llm::complete`, `kv::get`/`set`,
    `identity::claims`, `secrets::get`/`reveal`
  - Error types with `thiserror`; async functions with `tokio`
  - `Cargo.toml` with `wasm32-wasip2` target support
  - Unit tests with mock host functions
  - `README.md` with quick-start example
  - Gate: `cargo check -p flint-skill --target wasm32-wasip2`; unit tests pass

- **G3 — p14-c003-a2ui-hot-reload:**
  When a DESIGN.md or catalog change is detected, re-compile the A2UI component
  catalog via `StateManager::do_compile()` and notify connected AG-UI SSE clients.
  - Add a file-watcher or DB-trigger-based change detector for `flint_a2ui` tables
  - Wire the existing `StateManager` re-compile path to emit an AG-UI event
  - Client SDK `useFlintRegistry()` should auto-refresh on receiving the event
  - Gate: integration test proving a catalog change triggers a re-compile + SSE push

### P2 — Ship if capacity allows

- **G4 — p14-c004-jwt-rotation-automation:**
  `scripts/rotate_staging_jwt.sh` — regenerates `secrets/jwt_secret.txt` and
  updates the GitHub Actions secret via `gh secret set STAGING_JWT_SECRET`.
  - 30-line shell script; reads existing secret, generates new one, updates both
  - Document in `docs/runbook.md`
  - Gate: `bash -n` passes; dry-run mode (`--dry-run`) works

- **G5 — p14-c005-kiln-metrics:**
  Add Kiln-specific Prometheus counters to `fke-server`:
  - `kiln_invocations_total{function}` — counter by function name
  - `kiln_fuel_consumed_total` — histogram of fuel consumed per invocation
  - `kiln_epoch_traps_total` — counter of epoch deadline traps
  - Gate: `cargo test -p fke-server` passes; counters visible on `/metrics`

### Risk areas

- **sqlx 0.9 upgrade** (G1): pgvector 0.4.2 supports sqlx `>=0.8, <0.10`. Upgrading
  the workspace to `0.9` should work but needs verification — the `Encode`/`Type`
  trait impls must match.
- **A2UI hot-reload** (G3): The `StateManager` already has a `do_compile()` path.
  The new work is the change detector + SSE notification plumbing.
- **flint-skill crate** (G2): Targeting `wasm32-wasip2` requires the WIT bindings
  generated by `cargo component`. The crate is consumer-side only; no host changes.
