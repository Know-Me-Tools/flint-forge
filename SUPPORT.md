# Support

This document is for **non-security** questions, bugs, and usage help. For
vulnerabilities, see [SECURITY.md](SECURITY.md) instead.

## Getting Help

- **Bug reports and feature requests** — open a [GitHub
  issue](https://github.com/Know-Me-Tools/flint-forge/issues). Include your
  Flint Forge version, Postgres version, and enough detail to reproduce.
- **Questions about usage or configuration** — open a [GitHub
  Discussion](https://github.com/Know-Me-Tools/flint-forge/discussions) if
  enabled, or a GitHub issue tagged `question` otherwise.
- **Documentation** — start with [README.md](README.md) and `docs/`,
  especially `docs/FLINT-FORGE-SPEC.md` for the architecture and design
  contracts.

## What to Expect

Flint Forge is self-hosted, community-supported software: there is no
commercial support contract and no guaranteed response time. Maintainers
triage issues as time allows. Well-scoped bug reports with a minimal
reproduction are the fastest path to a fix.

## Reporting a Security Issue

Do not use GitHub issues for vulnerabilities. See
[SECURITY.md](SECURITY.md) for the private disclosure process.
