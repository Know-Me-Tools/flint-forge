## Type Design Decisions

| WIT Type | Represents | Why |
|---|---|---|
| `list<string>` | JSON row array from DB | WIT has no native `json` type; JSON is encoded at the host boundary |
| `string` | JSON object (opts, claims) | Same reason — JSON-encoded at boundary |
| `list<u8>` | Arbitrary bytes in KV | Raw bytes; caller encodes/decodes |
| `record host-error` | Typed error in each interface | Must be defined inside the interface that uses it (WIT scoping rule) |
| `resource secret` | Opaque secret handle | Prevents raw value from crossing WASM boundary except through `reveal` |
| `option<string>` | Optional JWT / model name | Standard WIT optional |
| `result<T, host-error>` | Fallible host calls | Standard WIT result |

## Freeze Rules

- **No backward-incompatible changes** to frozen interfaces or the `edge-function` world without bumping to `flint:host@0.2.0`.
- **Additive changes** (new interfaces) require a new `use` in the world and bumping the minor version.
- **The `secrets` interface's `resource secret` must remain opaque** — no direct field access, only `reveal` for Cedar-gated, audited disclosure.
- **`host-error` must not carry JWT payloads, claim values, tenant IDs, or secret material** — only a code and human-readable message.
